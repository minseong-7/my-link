# 마이링크 (MyLink) - 제품 요구사항 정의서 (PRD)

## 1. 프로젝트 개요

- **프로젝트명**: 마이링크 (MyLink)
- **목적**: 단일 웹페이지에서 개인의 다양한 링크와 작업물을 모아 직관적으로 보여주는 멀티 링크 프로필 서비스 구축.
- **대상 사용자**: 자신의 포트폴리오, 블로그, SNS, 작업물 등을 한 곳에 모아 효과적인 퍼스널 브랜딩을 하고 싶은 크리에이터 및 일반 사용자.

---

## 2. 핵심 기능 목록 (MVP 중심)

| 분류 | 기능명 | 설명 |
|---|---|---|
| 인증 | 구글 소셜 로그인 | Firebase Auth를 이용한 구글 계정 기반 간편 회원가입 및 로그인 단독 지원 |
| 프로필 관리 | 프로필 인라인 편집 | 이미지 업로드 기능 제외(구글 사진만 연동). 이름(username), 디스플레이 네임(displayName)과 소개글(Bio)을 **인라인 편집**으로 수정 가능 |
| DB 구조 | 서브 컬렉션 모델링 | Firebase Firestore 사용. 사용자 도큐먼트 내에 서브 컬렉션을 생성하여 링크 데이터 관리 |
| 링크 관리 | 링크 인라인 CRUD | 팝업이나 별도 화면 없이 내용(제목, URL)을 클릭해 그 자리에서 즉시 수정하는 **인라인 편집** 허용 |
| 링크 UI | 구글 파비콘 API 사용 | 수동 이미지 등록 없이, 구글 Favicon API를 연결하여 링크 도메인의 아이콘 자동 추출/표시 |
| 디자인 | 테마 기능 제거 | 스킨/테마/다크모드 커스텀 등 꾸미기 요소를 완전히 배제하고 고정된 단일 디자인 사용 |
| 퍼블릭 페이지| 디스플레이 네임 기반 깔끔한 URL | 별도의 영문 핸들 아이디나 '@' 기호 없이, 사용자 **디스플레이 네임** 자체가 고유 접속 경로(`url.com/displayName`)가 됨 |

> **🚀 참고 (향후 추가 예약 기능)**: 초기 MVP 개발 완료 후, 별도 버전업을 통해 링크의 '클릭 수 조회' 통계 기능만 추가 예정.

## 3. 시스템 상세 설명

### 3.1 디스플레이 네임 기반 라우팅 및 프로필(인라인)
- **디스플레이 네임 URL 및 예약어 처리**: 구글 지메일 아이디의 앞부분을 가져와 값으로 설정하는 디스플레이 네임(displayName)이 곧 퍼블릭 주소(`url.com/displayName`)가 되므로 라우팅이 매우 직관적이다. 단, 시스템 주소와의 충돌을 막기 위해 가입(또는 수정) 시 `login`, `dashboard`, `admin`, `api` 등의 라우트 키워드는 **사용 불가 예약어(금칙어)**로 처리하여 생성을 막는다.
- **프로필 정보 수정 (인라인)**: 설정 창으로 이동하지 않고, 프로필 메인 화면에서 이름(username), 디스플레이 네임(displayName) 텍스트나 소개글 텍스트를 클릭하면 즉시 입력창(input)으로 변환되어 인라인 수정이 가능토록 구현한다.
- **이미지 고정**: 사진 업로드/변경 기능은 원천 배제하며, 구글 로그인 시 반환되는 프로필 사진 URL만 화면에 고정 출력한다.

### 3.2 데이터베이스(Firestore) 설계 가이드
- 구글 지메일 계정의 앞부분을 추출한 `displayName`(URL 식별키/슬러그), 실제 사용자 이름인 `username`, 구글 프로필 이미지 URL, 소개글 등을 메인 문서(`users/{uid}`)에 저장한다.
- 관리의 효율성을 위해 사용자의 메인 문서 하위에 `links` **서브 컬렉션(Subcollection)**을 생성하여 각각의 링크 항목을 별도의 문서로 취급한다.

### 3.3 링크 대시보드 제어 (단순화 극대화)
- **인라인(Inline) 포맷 채택**: 링크를 추가한 뒤, 제목과 URL을 수정할 때 모달(Modal) 창을 띄우지 않고, 화면에 그려져 있는 텍스트를 클릭하여 바로 편집하는 방식 채택.
- **제외 기능 확정**: 사용하지 않을 비활성화(토클) 버튼과, 순서를 바꿀 수 있는 상하 드래그 앤 드롭 기능을 개발 영역에서 완전히 지운다.
- **파비콘 API 접목**: 구글 파비콘 API(`https://s2.googleusercontent.com/s2/favicons?domain=...`)를 직접 호출하여 별도의 서버 가공 없이 링크별 도메인 썸네일을 손쉽게 확보한다.

### 3.4 퍼블릭 페이지 뷰
- **테마리스(Themeless) 단일 뷰**: 테마 기능이 완전히 제거되었으므로, 하나의 완성된 깔끔한 레이아웃만 서비스 전체에 강제/적용한다.
- 모바일 화면용 미리보기(Preview Frame) 컴포넌트는 제작하지 않는다. 현재 본인이 컨트롤하고 있는 뷰 자체가 퍼블릭 페이지의 모습과 거의 일치하도록 직관적으로 설계한다.
