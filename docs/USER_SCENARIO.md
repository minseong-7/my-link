# 1. 방문자 (Visitor) 시나리오
방문자는 링크 소유자가 공유한 마이링크 페이지를 열람하고, 등록된 외부 링크로 이동하는 주체입니다.

### 1.1. 페이지 방문 및 열람
* 사용자는 [소유자의 프로필 정보를 확인]하기 위해 [공유받은 URL(`mylink.com/displayName`)로 접속]한다.
* System: 유효한 URL인 경우 해당 사용자의 프로필(사진, 이름, 소개)과 링크 목록을 로드하여 보여준다.

### 1.2. 링크 이동
* 사용자는 [관심 있는 외부 페이지(블로그, 포트폴리오 등)를 보기] 위해 [해당 링크 카드를 클릭]한다.
* System: 새 탭에서 해당 URL을 연다. (추후 이 시점에 클릭 이벤트를 카운팅한다.)

# 2. 소유자 (Owner) 시나리오

### 2.1. 회원가입 및 로그인 (Onboarding)
* 사용자는 [나만의 링크 페이지를 만들기] 위해 [메인 화면의 "Google로 시작하기" 버튼을 클릭]한다.
* System: Google 로그인 팝업을 띄우고 인증을 진행한다.
* 사용자는 [서비스 이용을 시작]하기 위해 [Google 계정 연동을 승인]한다.
* System: 
    * 신규 유저: Google 프로필(이름, 이메일, 사진) 정보를 가져와 계정을 생성하고 초기 대시보드로 이동시킨다.
    * 기존 유저: 대시보드로 즉시 이동시킨다.

### 2.2. 프로필 관리 (Profile Management) - 인라인 편집
* 사용자는 [자신의 표시 이름(Username)을 수정]하기 위해 [프로필 영역의 이름 텍스트를 클릭하고 내용을 입력]한다.
* System: 텍스트 필드를 수정 모드로 전환하고, 포커스가 해제되면 변경 사항을 자동 저장한다.
* 사용자는 [자신의 고유 URL ID(DisplayName)를 변경]하기 위해 [프로필 영역의 @아이디 텍스트를 클릭하고 새 ID를 입력]한다.
* System: 입력된 ID의 중복 여부를 체크하고, 사용 가능하면 업데이트한다. (실패 시 에러 표시)
* 사용자는 [자신을 소개]하기 위해 [Bio(소개글) 영역을 클릭하고 내용을 작성]한다.
* System: 빈 값이어도 클릭 가능한 영역(placeholder)을 제공하며, 입력 내용을 저장한다.

### 2.3. 링크 관리 (Link Asset Management)
* 사용자는 [새로운 포트폴리오나 소셜 링크를 추가]하기 위해 ["+새 링크 추가" 버튼을 클릭]한다.
* System: 리스트 하단(또는 상단)에 비어있는 새로운 링크 카드를 생성한다.
* 사용자는 [링크의 제목을 설정]하기 위해 [링크 카드의 제목 텍스트 부분을 클릭하고 입력]한다.
* 사용자는 [연결될 웹사이트 주소를 설정]하기 위해 [링크 카드의 URL 텍스트 부분을 클릭하고 주소를 입력]한다.
* System: 입력된 URL의 도메인을 파악하여 파비콘 아이콘을 자동으로 갱신해 보여준다.
* 사용자는 [더 이상 필요 없는 링크를 정리]하기 위해 [링크 카드의 삭제(휴지통) 아이콘을 클릭]한다.
* System: 사용자에게 확인(Confirm) 절차 없이(또는 간단한 확인 후) 즉시 리스트에서 해당 링크를 제거한다.

### 2.4. 계정 관리
* 사용자는 [서비스 이용을 마치거나 계정을 보호]하기 위해 [로그아웃 버튼을 클릭]한다.
* System: 세션을 종료하고 메인 랜딩 페이지로 이동시킨다.
